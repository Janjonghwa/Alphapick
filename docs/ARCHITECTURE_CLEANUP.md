# 아키텍처 정리 내역

## 1. 목적

본 문서는 AlphaPick 코드베이스를 현재 주식 분석 서비스 구조에 맞게 정리한 내역을 기록합니다.

## 2. 정리 방향

- 주식 분석과 관련 없는 레거시 요소 제거
- 포트폴리오 추천 정책을 성향별 허들 기반으로 정규화
- 기본 대시보드와 오늘의 포트폴리오 전체 화면의 역할 분리
- 사용자 화면에서 백테스트 메뉴와 페이지 제거
- 네이비/민트 기반 금융 대시보드 UI로 정리

## 3. 현재 화면 구조

| 경로 | 역할 |
|---|---|
| `/` | 기본 대시보드 홈 |
| `/portfolio` | 오늘의 포트폴리오 편입 종목 전체 |
| `/stocks` | 종목 검색 |
| `/stocks/{ticker}` | 종목 리포트 |
| `/community` | 커뮤니티 |
| `/login`, `/register`, `/mypage` | 사용자 기능 |

## 4. 포트폴리오 정책 정규화

- `total_score` 단일 기준이 아니라 회사 점수, 타이밍 점수, 신뢰도 점수를 함께 확인한다.
- 중립형, 공격형, 안정형에 따라 서로 다른 허들을 적용한다.
- 편입 종목 수와 시장 상태를 반영해 현금 비중을 계산한다.
- 섹터 쏠림이 발생하면 초과 비중을 재분배하거나 현금으로 전환한다.

## 5. 프론트엔드 정리

- AlphaPick 로고는 기본 대시보드 홈(`/`)으로 이동한다.
- 사이드바의 오늘의 포트폴리오는 `/portfolio`로 이동한다.
- 기본 대시보드에서는 시장 지표와 편입 종목 상위 15개를 보여준다.
- 오늘의 포트폴리오 화면에서는 편입 종목 전체를 보여준다.
- 백테스트 화면과 메뉴는 사용자 화면에서 제거했다.

## 6. 검증

- Django 설정 검사: `manage.py check`
- 프론트엔드 빌드: `npm run build`
- 주요 화면 응답: `/`, `/portfolio`, `/stocks`
